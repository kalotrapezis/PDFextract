# Academic-DB — Εργασία με Βάση Δεδομένων Paper (Human-in-the-Loop)

## Βασική Αρχή

Skill **συνεργασίας**: το AI δουλεύει, ο χρήστης ελέγχει/επαληθεύει/εγκρίνει σε κάθε
βήμα. Κανένα βήμα δεν ξεκινά πριν ολοκληρωθεί και επιβεβαιωθεί το προηγούμενο.

**Τι διορθώνει σε σχέση με το παρελθόν (τα δύο παράπονα):**
1. **Καλύτερη εύρεση πηγών** — αντί για γενικό web search, χρησιμοποιεί τον
   **connector του Consensus** (μηχανή έρευνας βασισμένη σε αποδείξεις) και αξιολογεί
   σοβαρά κάθε πηγή.
2. **Αποσπάσματα από ΟΛΟ το paper** — η εξαγωγή γίνεται **ενότητα-ενότητα** από τη
   βάση (Εισαγωγή, Μέθοδος, Αποτελέσματα, Συζήτηση), **ΟΧΙ μόνο από την περίληψη**.
   Η βάση κρατά **κάθε γραμμή**, οπότε καμία ενότητα δεν παραλείπεται.

**Μία βάση ανά εργασία.** Κάθε εργασία ζει στον δικό της φάκελο με δικό της
`papers.db`. Δεν ανακατεύονται paper διαφορετικών εργασιών.

**Σειρά βημάτων:** 0 → 1 → 1β → 2 → 3 → 4 → 5 → 6 → 7 → 8

## ΠΩΣ ΧΤΙΖΕΤΑΙ Η ΒΑΣΗ (μην το ξαναψάχνεις)

**Η βάση `papers.db` είναι η ΜΟΝΗ πηγή αλήθειας — ΑΝΤΙΚΑΘΙΣΤΑ το `ΑΠΟΣΠΑΣΜΑΤΑ.md`.**
Δεν φτιάχνουμε πλέον markdown με αποσπάσματα· όλα (σκελετός + σχόλια + αποσπάσματα)
ζουν μέσα στη βάση.

**Σχήμα (το φτιάχνει μόνο του το `db.py` στο πρώτο τρέξιμο):**
- `papers(id, filename, title, json_path, n_pages, authors, year, journal, doi, apa)` ← **+βιβλιογραφία**
- `sections(id, paper_id, seq, title, page_start, page_end)`
- `blocks(id, paper_id, section_id, page_no, seq, block_type, text, html)` ← **κάθε γραμμή**
- `annotations(section_id UNIQUE, who_speaks, whose_paper, importance, summary, tags)`
- `excerpts(id, paper_id, section_id, page_no, cite_id, text, relevance, used, citation)` ← **αποσπάσματα**
- `blocks_fts` (FTS5) για αναζήτηση **ακριβών λέξεων**
- `vec_sections` + `section_embed_meta` (sqlite-vec, BAAI/bge-m3) ← **σημασιολογική** αναζήτηση (προαιρετικό)

**Χτίσιμο (με τη σειρά):**
```bash
cd <φάκελος εργασίας>
export PAPERSDB="$(pwd)/papers.db"                  # ΜΙΑ βάση ανά εργασία
"$PY" "$PH/convert.py" papers/*.pdf                 # 1) PDF -> JSON (δίπλα στα PDF)
"$PY" "$PH/ingest.py"  papers/*/*.json              # 2) JSON -> papers.db (σκελετός + κάθε γραμμή)
"$PY" "$PH/embed.py"                                # 2β) (προαιρετικό) σημασιολογική ενσωμάτωση ενοτήτων
# 3) annotations + excerpts ανά ενότητα (Βήμα 3)
# 4) query.py για ανάγνωση κατά τη συγγραφή
```

**Εντολές βάσης (cheat-sheet):**
```bash
"$PY" "$PH/query.py" papers                         # λίστα paper
"$PY" "$PH/query.py" sections <paper_id>            # ΟΛΕΣ οι ενότητες (+annotations)
"$PY" "$PH/query.py" section <section_id>           # πλήρες κείμενο ενότητας
"$PY" "$PH/query.py" search "όρος"                  # full-text (ακριβείς λέξεις) σε όλα τα paper
"$PY" "$PH/query.py" semantic "ιδέα/ερώτημα"        # σημασιολογικά (κατά νόημα) — ενότητες, score
"$PY" "$PH/query.py" hybrid "ιδέα/ερώτημα"          # συνδυασμός λέξεων+νοήματος (RRF) — προτεινόμενο
"$PY" "$PH/embed.py" --status                       # πόσες ενότητες είναι ενσωματωμένες
"$PY" "$PH/query.py" todo                           # ενότητες ΧΩΡΙΣ annotation (στόχος: 0)
"$PY" "$PH/query.py" excerpts [--paper N] [--unused]# αποσπάσματα
"$PY" "$PH/query.py" bibliography                    # στοιχεία πηγών (+missing)
"$PY" "$PH/annotate.py" set <sid> --who .. --whose .. --importance .. --summary ..
"$PY" "$PH/annotate.py" excerpt <sid> --text "ακριβές" --relevance ".." --cite-id A001
"$PY" "$PH/annotate.py" use <excerpt_id> --citation "(Smith, 2019, σ.5)"
"$PY" "$PH/annotate.py" bib <paper_id> --authors .. --year .. --journal .. --doi .. --apa ..
```

## Βήμα 3 — Annotations & Αποσπάσματα ΕΝΟΤΗΤΑ-ΕΝΟΤΗΤΑ (διόρθωση παραπόνου #2)

> 🤖 **Μοντέλο: claude-opus-4-8** (effort: high) — βαθιά ανάγνωση όλων των ενοτήτων

**Ο κανόνας:** Διαβάζεις **ΟΛΕΣ τις ενότητες κάθε paper**, όχι μόνο την περίληψη.
Το `query.py todo` εγγυάται ότι καμία δεν ξεχνιέται.

**Ροή ανά paper:**
```bash
"$PY" "$PH/query.py" sections <paper_id>     # ΟΛΕΣ οι ενότητες
"$PY" "$PH/query.py" section <section_id>    # πλήρες κείμενο μιας ενότητας
```
Για **κάθε** ενότητα (Εισαγωγή, Θεωρία, Μέθοδος, **Αποτελέσματα**, **Συζήτηση**,
Συμπεράσματα — όχι μόνο Abstract):
1. Διάβασε το πλήρες κείμενο.
2. **Annotation** στη βάση:
   ```bash
   "$PY" "$PH/annotate.py" set <section_id> \
     --who "ποιος μιλά (οι συγγραφείς; παραθέτουν άλλον;)" \
     --whose "ποιανού η ιδέα — ΑΝ είναι άλλου, γράψε ΠΡΩΤΟΤΥΠΟ συγγραφέα+έτος π.χ. 'Bandura, 1977'" \
     --importance "πού χρησιμεύει στη ΘΕΣΗ" --summary "2 γραμμές"
   ```
   ⚠️ Το `whose_paper` είναι κρίσιμο για τις **δευτερεύουσες αναφορές** (Βήμα 5): αν η
   ενότητα αποδίδει ιδέα σε άλλον ερευνητή, κατέγραψε εκεί τον πρωτότυπο (όνομα+έτος).
   (Πολλά μαζί: `annotations.json` = λίστα `{section_id, who_speaks, whose_paper,
   importance, summary}` και `annotate.py load annotations.json`.)
3. **Αξιοποιήσιμο απόσπασμα → στη ΒΑΣΗ** (ακριβές αντίγραφο — καμία παράφραση·
   σελίδα/ενότητα μπαίνουν αυτόματα):
   ```bash
   "$PY" "$PH/annotate.py" excerpt <section_id> \
     --text "ακριβές κείμενο όπως στο PDF" \
     --relevance "σε ποια θέση/ερώτημα χρησιμεύει" --cite-id A001
   ```
   **ΟΧΙ `ΑΠΟΣΠΑΣΜΑΤΑ.md`** — τα αποσπάσματα ζουν στον πίνακα `excerpts`.

**Έλεγχος πληρότητας:** `query.py todo` → **πρέπει να μείνει 0** ενότητες χωρίς
annotation. Αν μένει, την αγνόησες — γύρνα πίσω.

**Διασταύρωση:** συμφωνία/αντίθεση paper → στο `importance`.

**Έξοδος:** annotations σε ΟΛΕΣ τις ενότητες + αποσπάσματα στη βάση. Σύνοψη με
`query.py excerpts` (κατανομή ανά ενότητα — να φαίνεται ότι ΔΕΝ είναι μόνο Abstract).
Ενημέρωση MAP.md.

> ⛔ Δεν προχωράς στο 4 χωρίς ρητή επιβεβαίωση χρήστη.

## Βήμα 5 — Συγγραφή Εργασίας (ενότητα-ενότητα από τη βάση)

> 🤖 **Μοντέλο: claude-opus-4-8** (effort: high) — υψηλή ποιότητα ακαδημαϊκής γραφής

**Πριν ξεκινήσεις:** διάβασε `/mnt/skills/public/docx/SKILL.md` για σωστό .docx.

Γράψε με **«Τρένο της Σκέψης»** (κάτω). Αντλείς **από τη βάση**: `query.py excerpts`,
`query.py search "όρος"`, `query.py section <id>` → **κάθε αναφορά με σωστή σελίδα/ενότητα**.

**Όταν χρησιμοποιείς ένα απόσπασμα, μαρκάρισέ το:**
```bash
"$PY" "$PH/annotate.py" use <excerpt_id> --citation "(Smith, 2019, σ.5)"
```
Έτσι το Βήμα 7 ξέρει αυτόματα τα αχρησιμοποίητα.

- **Εξώφυλλο** (στοιχεία Βήματος 0), page break, αρίθμηση από σελ. 2.
- **APA 7** εντός κειμένου· σελίδα `σ. Χ` **μόνο** σε αυτούσιες παραθέσεις.

### ΚΑΝΟΝΑΣ ΑΝΑΦΟΡΩΝ — σωστή απόδοση & δευτερεύουσες (ΚΡΙΣΙΜΟ)

**Απόδωσε την ιδέα σε ποιον ανήκει, όχι απλώς σε ποιο paper τη διάβασες.** Η βάση το
ξέρει: `annotations.who_speaks` (ποιος μιλά) και `annotations.whose_paper` (ποιανού η ιδέα).

- **Πρωτογενής:** η ιδέα είναι των ίδιων των συγγραφέων του paper που διάβασες →
  `(Bègue et al., 2017)`. Το paper είναι στη βάση/βιβλιογραφία.
- **Δευτερεύουσα / έμμεση πηγή (APA «as cited in»):** το paper που διάβασες παραθέτει/
  συνοψίζει ΑΛΛΟΝ ερευνητή (το `whose_paper` δείχνει άλλον). Σωστή μορφή APA 7:
  - Παρενθετικά: `(Πρωτότυπος, Έτος, όπ. αναφ. στο Διαβασμένο, Έτος)`
    π.χ. `(Bandura, 1977, όπ. αναφ. στο Bègue et al., 2017)`.
  - Αφηγηματικά: `Ο Bandura (1977, όπ. αναφ. στο Bègue et al., 2017) υποστηρίζει…`
  - **«όπ. αναφ. στο»** = η ελληνική συντομογραφία του «as cited in» (ΟΧΙ ολόκληρο
    «όπως αναφέρεται στο»). Αν δεν ξέρεις το έτος του πρωτότυπου, παρέλειψέ το.
  - **Χωρίς σελίδα** σε παράφραση· `σ. Χ` μόνο αν είναι **αυτούσια παράθεση**.
  - **Στη Βιβλιογραφία μπαίνει ΜΟΝΟ το paper που διάβασες** (Bègue, 2017) — **ΟΧΙ** ο
    Bandura, γιατί δεν τον διάβασες στο πρωτότυπο.
  - Αν ο πρωτότυπος είναι σημαντικός και βρίσκεται εύκολα, **προτίμησε** να τον βρεις/
    προσθέσεις ως δική του πηγή (Βήμα 1) και να τον αναφέρεις πρωτογενώς.

**Πρακτικά:** στο Βήμα 3, όταν μια ενότητα αποδίδει ιδέα σε άλλον, γράψε στο
`whose_paper` τον **πρωτότυπο συγγραφέα + έτος** (π.χ. «Bandura, 1977»). Έτσι εδώ ξέρεις
αν η αναφορά είναι πρωτογενής ή δευτερεύουσα και τη γράφεις σωστά.

- **Word comments (TK)** για κάθε αναφορά — **ίδια τεχνική με το `academic-el`**
  (Στάδιο Α: `splitP()` ώστε κάθε αναφορά σε δικό της `<w:r>`· Στάδιο Β: εισαγωγή σχολίων
  με lxml, styles "aa"/"a9", `comments.xml`, rels, Content_Types). Περιεχόμενο σχολίου:
  Αρχείο / Ενότητα / Σελίδα / Απόσπασμα (από τη βάση).
- **Όριο λέξεων:** στόχευσε στο εύρος· ανέφερε τελικό αριθμό.

**Έξοδος:** `.docx` με comments TK + αριθμός λέξεων. Ενημέρωση MAP.md.

> ⛔ Δεν προχωράς στο 6 χωρίς επιβεβαίωση ότι ο χρήστης διάβασε.

## Βήμα 8 — Τελικός Έλεγχος Αναφορών & DOI

> 🤖 **Μοντέλο: claude-opus-4-8** (effort: max) — η τελική πύλη ποιότητας

Διάβασε `query.py excerpts`, **`query.py bibliography`** (στοιχεία πηγών από τη βάση),
το τελικό `.docx`. Έλεγξε:
- **Ορφανές αναφορές vs δευτερεύουσες (ΚΡΙΣΙΜΟ):** για κάθε `(Συγγραφέας, Έτος)` στο κείμενο:
  - ταιριάζει με `authors`/`year` paper της βάσης → **πρωτογενής, ΟΚ**·
  - ΔΕΝ ταιριάζει → πρέπει να είναι **δευτερεύουσα** με `όπ. αναφ. στο <paper της βάσης>`
    και αυτό το paper να υπάρχει στη βάση. Αν είναι σκέτο `(Bandura, 1977)` χωρίς
    «όπ. αναφ. στο …» και ο Bandura δεν είναι στη βάση → **⚠️ ΟΡΦΑΝΗ/ΛΑΘΟΣ ΑΠΟΔΟΣΗ**.
- **Ορφανές πηγές:** paper στη βάση που δεν αναφέρεται πουθενά στο κείμενο.
- **Σωστή απόδοση:** διασταύρωσε με τα `who_speaks`/`whose_paper` της βάσης ότι κάθε ιδέα
  αποδίδεται στον πραγματικό της δημιουργό (όχι απλώς στο paper όπου διαβάστηκε).
- **Διασταύρωση συγγραφέα/έτους:** κάθε πρωτογενής `(Συγγραφέας, Έτος)` ταιριάζει με
  `authors`/`year` της βάσης.
- **`missing`:** καμία πηγή δεν έχει κενό authors/year/doi/apa (`query.py bibliography`).
- **Αντιστοίχιση Word comments** ↔ excerpts (αρχείο/σελίδα/ενότητα από τη βάση).
- **Επαλήθευση DOI** με web_search (τίτλος/συγγραφείς/έτος/περιοδικό).
- **Ακαδημαϊκή ποιότητα** (Τρένο Σκέψης, παραθέσεις με σχολιασμό, λεξιλόγιο, όριο λέξεων).

Το **κεφάλαιο Βιβλιογραφία** της εργασίας παράγεται από τη βάση (`query.py bibliography`
→ πεδίο `apa`), όχι από ξεχωριστό αρχείο.

Δώσε **Τελική Βαθμολογία Ετοιμότητας** (✅/⚠️ ανά κατηγορία) → «ΕΤΟΙΜΟ ΓΙΑ ΠΑΡΑΔΟΣΗ»
ή «ΧΡΕΙΑΖΕΤΑΙ ΔΙΟΡΘΩΣΗ — [σύνοψη]». Τελική ενημέρωση MAP.md.
